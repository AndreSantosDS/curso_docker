Para executar a imagem primeiramente vamos precisar fazer o build;
O comando é docker build <diretório da imagem>;
Depois vamos utilizar o docker run <imagem> para executá-la;

docker build .
o . representa a pasta do seu projeto na sua máquina.
Exemplo:
meu-projeto/
├── Dockerfile
├── package.json
├── app.js
└── routes/
O Docker pega essa pasta como contexto e começa a executar o Dockerfile linha por linha.
Depois ele chega em: WORKDIR /app
e cria (ou passa a usar) a pasta /app dentro do container.
Sua máquina
(meu-projeto)
        ↓
docker build .
        ↓
Docker lê o Dockerfile
        ↓
Cria container temporário
        ↓
WORKDIR /app
(cria /app dentro do container)
        ↓
COPY package*.json .
(copia para /app)
        ↓
COPY . .
(copia o resto do projeto para /app)
